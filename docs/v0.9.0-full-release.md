## 一、开发理念

### 1.1 为什么是 cdx-brain？

Codex 原生记忆系统有两个根本局限：
1. **热记忆不可控** — 全部对话无差别存入，没有质量筛选
2. **跨会话断层** — 每次新 session 都从零开始，不继承前序决策链

cdx-brain 的解决思路：**三层级记忆架构**，让记忆像人脑一样分层管理。

### 1.2 三层级哲学

```
┌──────────────────────────────────────────────┐
│         原生记忆（Codex extensions）           │
│     精炼 · 必需 · 会话启动自动加载             │
│     "只记住最重要的"                            │
├──────────────────────────────────────────────┤
│         热记忆（cdx-brain cache.db）           │
│     全部对话痕迹 · FTS5 全文检索 · 按需注入    │
│     "随时翻查所有聊天记录"                      │
├──────────────────────────────────────────────┤
│         共享底座（OpenViking）                 │
│     跨 Agent 语义检索 · 联邦记忆               │
│     "豹哥知道的事，康少也能查到"               │
└──────────────────────────────────────────────┘
```

### 1.3 v0.9.0 升级方向

从 v0.8.0 知识图谱 + 联邦共识出发，v0.9.0 补了三个痛点的"近亲能力"：

| 痛点 | 解决方案 |
|:----|:---------|
| Agent 总是重复踩同一个坑 | 反事实记忆：记录"放弃的方案+原因" |
| 没人知道记忆系统是否健康 | 哨兵监控：主动巡检而非被动等报告 |
| 跨会话不知道自己做到哪了 | 任务森林：跨会话任务 DAG + 用户画像 |

与其他 memory 方案核心差异：

| 对比维度 | cdx-brain | Mainline | COMPASS Skills |
|:--------|:---------:|:--------:|:--------------:|
| MCP 依赖 | ❌ 零 | ✅ 需要 | ✅ 需要 |
| 检索方式 | FTS5 + 语义 + RRF | 向量库 | ChromaDB |
| 自动晋升 | ✅ 门控评分 | ❌ 手动 | ❌ 手动 |
| 跨 Agent | ✅ OpenViking 联邦 | ❌ 单体 | ❌ 单体 |
| 零配置 | ✅ pip install 即用 | ❌ 需配置 | ❌ 需配置 |

## 二、方案选型

### 2.1 社区调研

2026-06-27 深入 Codex 社区三个项目：

**Mainline** — 反事实记忆
- 核心：Git notes 存"放弃的方案+原因"
- 启发：cdx-brain 晋升门控只存"被采纳的"，缺对应"为什么不做"通道
- 差距：6 个字段够用（id/subject/chosen/rejected/reason/context）

**Thread-Scoped Memory #15432** — 哨兵监控
- 核心：后台 Agent 持续监控记忆健康
- 启发：cdx-brain heartbeat 只做简单存活检查，缺主动质量巡检
- 差距：碎片率、孤立 session、反事实未归档等独有巡检项

**COMPASS Skills** — 任务 DAG + 用户画像
- 核心：跨会话任务依赖图 + 结构化画像
- 启发：cdx-brain session 按时间线存储，缺结构化任务关系
- 差距：DAG blocked_by + TTL、决策偏好画像

### 2.2 五决策点（豹哥评审）

| 决策 | 结论 | 理由 |
|:----|:----|:-----|
| 反事实自动捕获 | 默认开启 | 不记得等于没发生过；关键词收窄避免误报 |
| 哨兵巡检频率 | quick + deep 双通道 | 每次 store.py 结束跑 quick（<3s），凌晨 4:00 deep |
| 任务 DAG 粒度 | 任务级别 | 子步骤自动检测准确率太低 |
| 用户画像 | 这次就建 | 从决策偏好入手，不搞通用 preference |
| 实施优先级 | 4.1 → 4.3 → 4.2 | 反事实价值最高改动最小，任务森林最独立，哨兵需协调接口 |

## 三、核心能力（新增三大模块）

### 🥇 Phase 4.1 — 反事实记忆

**解决的问题**：Agent 踩过的坑，下次还会踩，因为没有"为什么不做"的记录。

**工作机制**：
```
对话结束 → store.py 提取 assistant_content
         → 匹配 [方案/方向/路径/改法] + [放弃/否决/不用/不行]
         → 否定语境守卫（防止"放弃午休写完了"误报）
         → 存入 counterfactuals FTS5 表
         → 下次同类问题触发 inject.py 反事实注入
```

**技术实现**：
- FTS5 + LIKE 双通道检索（解决 CJK 中文搜索）
- 正则触发词收窄：只有特定组合才触发
- 否定语境守卫：30 字符前缀检测 "没/别/不会/不能/不想/不要/还没/暂时"
- 数据模型：6 个字段（id/subject/chosen/rejected/reason/context/confidence...）

**CLI**：`cdx-brain cf add/list/search/stats`

### 🥈 Phase 4.3 — 任务森林

**解决的问题**：上次 Session 的进度、阻塞、下一站在哪——每次都说不清楚。

**数据模型**：
```
TaskNode(id, title, status, priority, blocked_by, tags, created_at, updated_at)
TaskEdge(source_id, target_id, relation)
UserProfile(tech_stack_preferences, architecture_style, decision_patterns, anti_patterns)
```

**关键设计**：
- 自动检测：只做 done 不做 in_progress（豹哥建议：done 语义确定性更强）
- blocked TTL：30 天未更新的 blocked 自动转 cancelled
- 用户画像从决策偏好入手（技术栈/架构/反模式/决策模式）
- Mermaid 可视化输出

**CLI**：`cdx-brain task add/status/block/list/graph/prune/stats/profile`

### 🥉 Phase 4.2 — 哨兵监控

**解决的问题**：记忆系统有没有问题——没人知道，直到用户报告说"我的记忆呢？"

**双通道巡检**：

| | Quick Check | Deep Check |
|:--|:-----------|:-----------|
| 耗时 | <3 秒 | 5-10 秒 |
| 触发 | 每次 store.py 结束 | 每日凌晨 4:00 |
| 检查项 | cache 大小/OV 连通性/BD 同步 | + 碎片率/孤立 session/反事实未归档 |
| 自动修复 | — | VACUUM |

**哨兵理念**：不只是报问题——能自动修的（如 VACUUM）自己修了再报，只有不可恢复的问题才告警。

**CLI**：`cdx-brain sentinel quick/deep/status`

## 四、实现细节

### 4.1 代码统计

- 新建 13 个文件，修改 3 个（schema.py / promote.py / cli.py）
- 总计 ~950 行代码（反事实 240 + 任务森林 370 + 哨兵 305）
- 全部通过 Python syntax check + 集成测试

### 4.2 自动串联到会话管线（最新）

4 个 hook 模板已更新，Phase 4 模块不再需要手动敲 CLI：

| Hook | 触发时机 | 新增行为 |
|:----|:---------|:---------|
| store.py | 每轮对话结束 | 自动提取反事实记忆 → 存入 counterfactuals 表 |
| inject.py | 用户输入开始时 | 反事实记忆作为第 6 源，RRF 融合排序 |
| compact.py | 上下文压缩后 | 自动触发哨兵 quick check |
| summary.py | 会话结束时 | 哨兵 quick check → 异常日志 |

### 4.3 独立性改造

- 所有环境变量从 CC_STAR_* 改为 CDX_BRAIN_*
- 数据目录从 .cc-star/ 改为 .cdx-brain/
- source 标签从 cc-star/ 改为 cdx-brain/
- 内存路径从 extensions/cc-star 改为 extensions/cdx-brain
- 零 cc-star 残留（全库 grep 验证通过）
