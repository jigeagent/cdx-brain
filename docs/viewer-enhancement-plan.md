> 2026-06-27 | 康少
> 覆盖 v0.5.0 → v0.9.0 全部能力 | 改动量：~680 行

---

## 一、设计原则

1. **一屏可扫** — 打开浏览器看到的就是系统全貌，不需要翻页
2. **空状态友好** — 数据还没积累时显示引导，不报错
3. **趋势优先** — 不展示"当前值"，展示"变化方向"
4. **零新依赖** — 后端 stdlib-only，前端 CDN 加载 Chart.js

## 三、8 张概览卡片

每张卡片 3 个核心指标，空状态显示 "--" 和引导语。

| 卡片 | 指标 1 | 指标 2 | 指标 3 |
|:----|:-------|:-------|:-------|
| 🗃️ 热记忆 | 总 traces | cold 数 | 本周新增 |
| 🧠 认知管线 | policies | skills | concepts |
| 🔗 知识图谱 | triples | 节点 | 关系类型 |
| ⛔ 反事实 | 总数 | 本月放弃 | 收敛趋势 |
| 📋 任务森林 | 活跃 | 阻塞 | 完成率 |
| 🌐 联邦记忆 | OV 状态 | 共享 Agent | 已同步 |
| 🏆 晋升门控 | 晋升数 | 7d 内晋升 | 拒绝率 |
| ❤️ 系统状态 | 哨兵状态 | cache 大小 | 上次巡检 |

卡片状态色：
- 🟢 全部正常
- 🟡 有警告（如 BD 同步超期、碎片率高）
- 🔴 有严重问题（cache > 1GB、OV 离线）

## 五、新增 API 一览

| 端点 | 用途 | 数据来源 |
|:----|:-----|:--------|
| `GET /api/dashboard` | 8 张概览卡片数据 | 聚合所有子模块 |
| `GET /api/cf/stats` | 反事实统计 | counterfactuals 表 |
| `GET /api/cf/timeline` | 反事实月度聚合 | counterfactuals 表 |
| `GET /api/tasks/stats` | 任务统计 | task_forest.json |
| `GET /api/tasks/active` | 活跃任务 | task_forest.json |
| `GET /api/tasks/mermaid` | Mermaid 源码 | task_forest.json |
| `GET /api/health/quick` | 哨兵 quick | sentinel/scout.py |
| `GET /api/health/history` | 健康趋势 | sentinel 历史 |
| `GET /api/search?q=xxx` | 六源检索 | RRF 融合 |
| `GET /api/pipeline` | 认知管线状态 | pipeline_state.json |
| `GET /api/activity` | 本周记忆活动 | traces 表 |

## 七、完整数据流

```
用户请求 /api/dashboard
     │
     ├── cache.db ───┬── traces COUNT/GROUP BY date
     │               ├── counterfactuals COUNT/GROUP BY subject
     │               └── triples COUNT
     │
     ├── pipeline_state.json ─── policies/skills/concepts
     │
     ├── task_forest.json ─── nodes/edges stats
     │
     ├── sentinel ─── run_quick_check()
     │
     └── user_profile.json ─── 可选
     │
     └── 聚合器合并 → 返回一个 JSON
```

所有数据源都是本地文件/SQLite，无外部依赖。
